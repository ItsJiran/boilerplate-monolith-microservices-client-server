# Database Schema Overview

This document provides an overview of the database schema for the application, organized by the owning module. The schema is designed to support the **Monolithic Modular** architecture.

## Core Structure and Field Details

### 1. Identity Module (Core)

#### `identity_users`
- `id`: `BIGINT UNSIGNED` - Primary key.
- `username`: `VARCHAR` - User's username.
- `email`: `VARCHAR` - User's email address, must be unique.
- `password`: `VARCHAR` - Hashed user password.
- `meta`: `JSONB` - JSON column for additional data/metadata (nullable).
- `userable_type`: `VARCHAR` - Part of the `userable` polymorphic relation (nullable).
- `userable_id`: `BIGINT UNSIGNED` - Part of the `userable` polymorphic relation (nullable).
- `timestamps`: `TIMESTAMP`

#### `identity_roles`
- `id`: `BIGINT UNSIGNED` - Primary key.
- `name`: `VARCHAR` - Role name.
- `slug`: `VARCHAR` - Unique slug for the role (unique).
- `timestamps`: `TIMESTAMP`

#### `identity_role_assignments`
- `id`: `BIGINT UNSIGNED` - Primary key.
- `user_id`: `BIGINT UNSIGNED` - Foreign key to `identity_users`.
- `role_id`: `BIGINT UNSIGNED` - Foreign key to `identity_roles`.
- `targetable_type`: `VARCHAR` - Scoped entity type (e.g., 'App\\Models\\Store').
- `targetable_id`: `BIGINT UNSIGNED` - Scoped entity ID.

#### `identity_access_tokens` (Sanctum)
- Standard Laravel Sanctum token structure.

### 2. Example Feature Module (e.g., Catalog)

#### `catalog_items` (Example Table)
- `id`: `BIGINT UNSIGNED` - Primary key.
- `type`: `VARCHAR` - Item type classification.
- `name`: `VARCHAR` - Entity name.
- `slug`: `VARCHAR` - Unique identifier/slug.
- `is_active`: `BOOLEAN` - Availability status.
- `meta`: `JSONB` - Additional properties.
- `timestamps`: `TIMESTAMP`

#### `catalog_categories` (Example Hierarchy)
- `id`: `BIGINT UNSIGNED` - Primary key.
- `parent_id`: `BIGINT UNSIGNED` - Tree structure parent.
- `name`: `VARCHAR` - Category name.
- `slug`: `VARCHAR` - URL slug.

### 3. Shared Module

#### `jobs`
- `id`: `BIGINT UNSIGNED`
- `queue`: `VARCHAR`
- `payload`: `LONGTEXT`
- Standard Laravel Job structure.

#### `notifications`
- `id`: `UUID`
- `type`: `VARCHAR`
- `notifiable_type`: `VARCHAR`
- `notifiable_id`: `BIGINT UNSIGNED`
- `data`: `TEXT`
- Standard Laravel Notification structure.

---

**Note:** Each module manages its own tables. Cross-module joins are **prohibited** within the application code, though tables reside in the same physical database.
